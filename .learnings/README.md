# Self-Improvement Skill for OpenClaw
## Continuous Learning and Error Capture

---

## Overview

This skill enables me to capture learnings, errors, and corrections for continuous improvement. It helps me learn from mistakes, user feedback, and new discoveries.

---

## When to Log Learnings

- [ ] A command or operation fails unexpectedly
- [ ] User corrects me with "No, that's wrong" or "Actually..."
- [ ] A requested capability doesn't exist
- [ ] External API or tool fails
- [ ] I identify outdated or incorrect knowledge
- [ ] A better approach is discovered for recurring tasks
- [ ] Before major tasks, to review previous learnings

---

## Learning Format

```markdown
### [YYYY-MM-DD] Learning Title

**Priority**: low | medium | high | critical  
**Status**: pending | resolved  
**Area**: frontend | backend | infra | tests | docs | config | trading | analysis

**Summary**: Brief description of what happened

**Context**: 
- When did this occur?
- What was I trying to do?
- What went wrong?

**Solution**: 
- How was it fixed?
- Code snippet if applicable

**Prevention**:
- How to avoid this in future?
- Checklist or pattern to follow

**Related**: Links to other learnings or files
```

---

## Quick Log Template

For rapid capture during work:

```markdown
### $(date +%Y-%m-%d) - [Brief Title]
**Priority**: [level] | **Status**: pending | **Area**: [category]
**Issue**: [What happened]
**Fix**: [How to fix/prevent]
```

---

## Review Checklist

Before starting major tasks:
- [ ] Review critical learnings
- [ ] Review high priority learnings
- [ ] Check for related past errors
- [ ] Apply prevention patterns
